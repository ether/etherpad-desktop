# Phase 3 — Bootstrap `@etherpad/mobile` (Capacitor scaffold) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land `packages/mobile/` as a Vite-built React app that mounts `<App />` from `@etherpad/shell` via a stub `CapacitorPlatform`, plus Capacitor's Android project. The shell boots, sees zero workspaces, opens the non-dismissable `AddWorkspaceDialog`. No real persistence — Phase 4 wires `@capacitor/preferences` + `@capacitor/filesystem`.

**Architecture:**
- `packages/mobile/` is a Vite app (not electron-vite — plain Vite + React).
- Capacitor 8 wraps the built static assets into an Android (and later iOS) WebView.
- `CapacitorPlatform` is the seam between the shared `@etherpad/shell` `Platform` interface and Capacitor's plugin APIs. Phase 3 ships a stub impl: empty state for reads, no-op for writes, no-op for events. Phase 4 swaps the storage parts for `@capacitor/preferences` / `@capacitor/filesystem`.

**Tech Stack:** Capacitor 8.x, Vite 8, React 19, TypeScript strict, pnpm workspaces. JDK 17 + Android Gradle Plugin 8.x (whatever the Capacitor 8 Android template ships with).

---

## Decisions locked in before this plan

1. **Capacitor 8 (latest stable).** Spec says "Capacitor 6+"; we use latest stable for the longest support runway.
2. **Vite, not electron-vite.** Mobile is a single-page React app served to a WebView. No main/preload split.
3. **Stub `CapacitorPlatform` for Phase 3.** Real Capacitor plugin wiring (preferences, filesystem, share, app, deeplinks) lands in Phase 4+. Phase 3 stays mechanical.
4. **Include the Capacitor-generated Android project in git.** This is the upstream pattern for Capacitor monorepos and what F-Droid expects to clone.
5. **Don't commit signing keys, gradle.properties secrets, or build outputs.** `.gitignore` handles the obvious ones.
6. **Browser smoke test in Phase 3.** Real Android emulator runs come in Phase 6 (permissions + deep links + share). For now we verify the bundle by loading it in headless Playwright Chromium under xvfb — that catches "did the shell mount? does it call platform.state.getInitial()? does the AddWorkspaceDialog open?"

## Task 2: Implement stub `CapacitorPlatform`

**Files:**
- Create: `packages/mobile/src/platform/capacitor.ts`

- [ ] **Step 1: Write the stub**

```typescript
import type { Platform } from '@etherpad/shell';

/**
 * Stub `CapacitorPlatform` for Phase 3. Returns empty state for every read
 * so the shell renders its first-launch UI (the non-dismissable
 * AddWorkspaceDialog). Write methods reject with NOT_IMPLEMENTED — Phase 4
 * replaces these with @capacitor/preferences + @capacitor/filesystem.
 *
 * Events are no-op subscribers: the unsubscribe fn does nothing because
 * nothing ever fires. Mobile is single-window so cross-process events aren't
 * meaningful; Phase 4 may add an in-process mitt bus if components want to
 * fire local events.
 */
export function createCapacitorPlatform(): Platform {
  const ok = { ok: true as const };
  const okValue = <T>(value: T) => ({ ok: true as const, value });
  const notImpl = (op: string) => Promise.reject(new Error(`[mobile/Phase 3] ${op} not implemented yet`));
  const noopUnsubscribe = (): (() => void) => () => {};

  return {
    state: {
      getInitial: async () =>
        okValue({
          workspaces: [],
          workspaceOrder: [],
          settings: {
            schemaVersion: 1 as const,
            defaultZoom: 1,
            accentColor: '#3366cc',
            language: 'en',
            rememberOpenTabsOnQuit: true,
            minimizeToTray: false,
            themePreference: 'auto' as const,
            userName: '',
          },
          padHistory: {},
        }),
    },
    workspace: {
      list: async () => okValue({ workspaces: [], order: [] }),
      add: () => notImpl('workspace.add'),
      update: () => notImpl('workspace.update'),
      remove: () => notImpl('workspace.remove'),
      reorder: () => notImpl('workspace.reorder'),
    },
    tab: {
      open: () => notImpl('tab.open'),
      close: () => notImpl('tab.close'),
      focus: () => notImpl('tab.focus'),
      reload: () => notImpl('tab.reload'),
      hardReload: () => notImpl('tab.hardReload'),
    },
    window: {
      setActiveWorkspace: async () => ok,
      reloadShell: () => {
        window.location.reload();
        return Promise.resolve(ok);
      },
      setPadViewsHidden: async () => ok,
      setRailCollapsed: async () => ok,
    },
    padHistory: {
      list: async () => okValue([]),
      pin: () => notImpl('padHistory.pin'),
      unpin: () => notImpl('padHistory.unpin'),
      clearRecent: () => notImpl('padHistory.clearRecent'),
      clearAll: () => notImpl('padHistory.clearAll'),
    },
    settings: {
      get: () => notImpl('settings.get'),
      update: () => notImpl('settings.update'),
    },
    httpLogin: {
      respond: () => notImpl('httpLogin.respond'),
    },
    updater: {
      checkNow: () => notImpl('updater.checkNow'),
      installAndRestart: () => notImpl('updater.installAndRestart'),
      getState: async () => ({ kind: 'unsupported', reason: 'mobile' }),
    },
    quickSwitcher: {
      searchPadContent: async () => [],
    },
    events: {
      onWorkspacesChanged: noopUnsubscribe,
      onPadHistoryChanged: noopUnsubscribe,
      onTabsChanged: noopUnsubscribe,
      onTabState: noopUnsubscribe,
      onSettingsChanged: noopUnsubscribe,
      onHttpLoginRequest: noopUnsubscribe,
      onUpdaterState: noopUnsubscribe,
      onPadFastSwitch: noopUnsubscribe,
      onMenuShellMessage: noopUnsubscribe,
    },
  };
}
```
